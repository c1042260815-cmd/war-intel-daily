# ⚔️ 美以伊战争情报日报系统

> 基于 KM 文章《腾讯版「龙虾」WorkBuddy 上线当天，我用它搭了一套行业情报日报系统》的五步情报方法论构建

## 📐 五步方法论（已完全内化）

本系统严格遵循文章第8点的五步方法论：

```
Step 1: 明确覆盖范围（关心哪些实体/信源/领域）
  ↓
Step 2: 定义排除规则（什么不要）
  ↓
Step 3: 规定信源标准（只要一手，三级分类 + 交叉验证）
  ↓
Step 4: 设定输出模板（8大板块，结构化、可行动）
  ↓
Step 5: 自动化分发（企微精简版 + HTML完整版 + JSON结构化）
```

详见 `intelligence-constitution.json`（情报宪法 — 系统的核心配置文件）。

## 📂 项目结构

```
war-intel-daily/
├── intelligence-constitution.json   # 📜 情报宪法（核心Prompt配置）
├── push.py                          # 📨 推送脚本（企微/飞书Webhook）
├── README.md                        # 本文件
└── reports/
    ├── 2026-03-30.html              # 📊 完整HTML日报（Day 30）
    └── 2026-03-30-summary.md        # 📱 企微精简版（≤4096字节）
```

## 📊 日报内容结构（8大板块）

| 板块 | 内容 | 示例 |
|------|------|------|
| **局势总览** | 100字当日核心态势 | 战争第30天升级概况 |
| **核心动态** | 3-5条重大事件+影响评估 | 以军空袭德黑兰120+弹药 |
| **军事行动** | 按行动方分类的详细军事动态 | 美以/伊朗/代理人/兵力部署 |
| **外交与政治** | 停火谈判、各方表态 | 伊朗6条 vs 美国15条条件 |
| **人道影响** | 伤亡数据、医疗系统 | 累计3200+人死亡 |
| **经济影响** | 油价、航运、供应链 | 铝供应/霍尔木兹海峡 |
| **风险信号** | 3级风险预警指标 | 地面入侵/航母对抗/核设施 |
| **累计数据** | Day N 关键数据汇总 | 战争30天全景数据 |

## 🔍 信源体系（三级分类 + 交叉验证）

- **Tier 1 一手信源**：新华社、央视、CNN、Reuters、AP、CENTCOM、IDF、IRGC 官方声明
- **Tier 2 权威分析**：ISW、GlobalSecurity.org、Just Security、各国智库
- **Tier 3 补充信源**：知乎深度、雪球分析（需 Tier 1 佐证）
- **规则**：所有军事行动信息必须至少两个独立来源交叉验证

## 🚀 使用方式

### 查看今日日报
直接在浏览器中打开 `reports/2026-03-30.html`

### 配置推送
```bash
# 设置环境变量
export WEWORK_WEBHOOK_URL="https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY"
export FEISHU_WEBHOOK_URL="https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_TOKEN"

# 运行推送
python push.py
```

### 扩展到其他领域
修改 `intelligence-constitution.json` 即可适配任何垂直领域：
- 将"美/以/伊"替换为你关心的实体
- 调整覆盖范围、排除规则、信源标准
- 保持输出模板结构不变

## 📝 方法论迁移指南

本系统的核心方法可迁移到任何情报领域：
1. **明确覆盖范围** — 你关心哪些厂商/实体/信源
2. **定义排除规则** — 什么不要（传言/情绪/历史回顾/重复）
3. **规定信源标准** — 只要一手（三级分类 + 交叉验证）
4. **设定输出模板** — 结构化、可行动（每条回答4个问题）
5. **自动化分发** — 推送到你的消费场景
